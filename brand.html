Brand

Canonical visual system for Project Lighthouse. Use this page as the source of truth when building new portal pages, editing existing ones, or producing any downstream asset. Any deviation is drift — check here first.

!
Absolute rules — non-negotiable
Never invent data · Never use off-brand colours · Never write content ungrounded in confirmed facts · Always audit every file for brand compliance before publishing · Accurate and honest only — flag gaps explicitly rather than fill them.

Build conventions
Technical rules for new portal pages
01Self-contained HTML — vanilla JS or React. No build steps, no external dependencies except Google Fonts.
02Apply background:#040E3D !important on both html and body. Also inline style="background:#040E3D" on the <html> tag for downloaded files.
03Never use color-mix() CSS — causes rendering failures.
04Body text: #F2F2F4 at full opacity only. Never partial opacity for body copy.
05Audit every file before publishing: grep -oEi '#071530|#8A9CC4|#A855F7|#EDF2FF|#F5F7FB|#F59E0B|color-mix' — zero matches expected.
06Callout Red #F65275 scoped to authority elements, gap indicators, and progress-bar starts — never decorative.
07All progress bars: #F65275 → #71F1F7 gradient, left to right.
08Hover backgrounds: #F2F2F4. Dark text on light: rgba(4,14,61,x).
Portal variants
Intentional divergence — not drift
Main portal (10 pages)
Dark-on-dark pattern. Navy background with translucent cards, teal accents, dark sidebar nav. Covers: index.html, portfolio.html, offering.html, scoring.html, actions.html, sales.html, gtm-assets.html, asset-roadmap.html, architecture.html, lighthouse-architecture.html.
Sales Portal · AE (standalone)
Intentionally different. Scrollable multi-section product-page pattern — alternating light and dark sections, white hero cards, full-bleed layout. Covers: sales-portal.html only. Do not propagate this pattern to main portal pages.
Sales View (bridge page)
Light theme exception within main portal. sales.html uses a light background (#F0F2F7) to bridge the dark ops portal with the white AE-facing content model — this is the only main portal page that does this.
Voice principles
How content reads across the portal
  • ✓
    Evidence-gated
    No credit without proof. Placeholder language only where data is genuinely absent — mark it "pending" or "awaiting", not invented.
  • ✓
    Direct and specific
    Short sentences. Concrete nouns. Numbers where meaningful. Flag what's confirmed vs what's pending.
  • ✓
    Honest about gaps
    Unverified states shown as "Unverified / Awaiting score" — never backfilled with invented values.
  • ✗
    No invented KPIs, placeholder scores, filler text
    If a number, label, colour, or fact isn't confirmed — stop. Ask. Flag the gap.
  • ✗
    No marketing copy
    This is an internal GTM tool, not a sales website. Labels come from the framework — not from persuasion.
  • ✗
    No invented customer names or proof points
    Reference customers must come from Crossfuze's published case studies or internally confirmed references. Stellantis is internal-only — confirm with Jes before naming externally.
