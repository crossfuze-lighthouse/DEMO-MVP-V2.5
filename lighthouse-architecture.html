Framework & Milestone Layer
13 subcategories across 3 dimensions
Strategic Alignment — 50% of score
1A Business Strategy Alignment (40%) · 1B ServiceNow Strategic Alignment (25%) · 1C Market Alignment (35%)
Commercial Readiness — 25% of score
2A Op Readiness & Scope (25%) · 2B GTM Collateral (20%) · 2C Sales & Demand Engine (25%) · 2D SN Partnership (15%) · 2E CS & Expansion (15%)
Delivery Expertise — 25% of score
3A Delivery Confidence (20%) · 3B History of Success (30%) · 3C Technical Expertise (25%) · 3D Thought Leadership (15%) · 3E Continuous Improvement (10%)
Milestone properties
Level (L1–L5)
Maturity level at which milestone should be evidenced. L1 = foundations. L2 = minimum to operate. L3 = externally defensible. L4/L5 = differentiating.
★ Priority (14 milestones)
Most important milestones at each level. Should be complete before claiming higher levels. Require verified evidence. IDs: 1B.4, 1B.10, 1B.11, 1C.1, 2A.3, 2A.9, 2A.16, 2B.2, 2C.6, 2C.12, 2E.13, 3B.8, 3B.9, 3B.14
Type: Evidence / Asset / Action
Evidence = verifiable proof (cert, record, reference, tracked data) · Asset = document/template built · Action = process actively running
Assessed objects
Offering (primary)
Specific service or solution. e.g. SMART ITSM, OOTBaaS CSM, Managed Admin. One score per offering per session.
Vertical (secondary)
Market focus. e.g. Financial Services, Public Sector. Assessed using same framework if needed.
Capability / Module (secondary)
Cross-cutting capability. e.g. AEO, Managed Services component. Scored independently.
Do not combine multiple offerings in one assessment. One session = one assessed object.

Scoring Engine — Single Maturity Score 1.00–5.00
Formula
Maturity Score = (SA × 0.50) + (CR × 0.25) + (DE × 0.25) SA = (1A×0.40) + (1B×0.25) + (1C×0.35) CR = (2A×0.25) + (2B×0.20) + (2C×0.25) + (2D×0.15) + (2E×0.15) DE = (3A×0.20) + (3B×0.30) + (3C×0.25) + (3D×0.15) + (3E×0.10) Output: 1.00 – 5.00 Level: 1.00–1.49 = L1 Ad Hoc 1.50–2.49 = L2 Developing 2.50–3.49 = L3 Standardized 3.50–4.49 = L4 Advanced 4.50–5.00 = L5 Best-In-Class
Scoring rules
Score the questions, not the milestones
Milestones give the assessor confidence to answer honestly. They don't mechanically produce the score.
Score lower when between levels
A 3 that is fully evidenced is more useful than a 4 that is not. Honest assessment = strategic value.
★ guides the score
If ★ milestones for a level are incomplete, reflect that in the score. Don't claim a level you can't evidence.
90-day cadence
Rescore every quarter. Previous scores pre-populate. Only update subcategories that materially changed.
Outputs produced
Maturity Score (1.00–5.00) + Level (L1–L5)
Single number. Drives Hub KPI strip, portfolio table, offering detail hero.
SA / CR / DE category scores
3 weighted sub-scores. Power portfolio heatmap columns and category cards in offering detail.
13 subcategory scores (1A–3E)
Individual 1–5 per subcategory. Power heatmap cells, offering detail breakdown, scoring session display.
Sales readiness flag
Ready / Ready with caveats / Not ready. Drives Sales View traffic light.
Priority actions (v2.0 milestone IDs)
Key gaps from scores + milestones. e.g. "Delivery playbook (2A.16) — OOTBaaS ITSM · L2 DE ★"

Operational Plan — Who does what and when
First scoring session
1
Brief function leads using stakeholder doc
Share Offerting_Maturity_Stakeholder_Assessment_Overview_3_4.docx. 30-min alignment call. Agree milestone ownership before scoring.
2
Start with SMART ITSM
Most mature offering. Builds confidence with the framework. Score all 13 subcategories. Allow 60–90 minutes.
3
Work subcategory by subcategory
Read the question. Pick 1–5 using anchors. Expand milestones to verify. Score lower when between levels.
4
Score remaining offerings in priority order
SMART ITSM → OOTBaaS ITSM → Managed Admin → SMART CSM → SMART HRSD → remaining 9.
Quarterly cadence (every 90 days)
W1
Review what changed in the quarter
New wins, new assets built, new SN engagements, team changes. Identify which offerings need rescoring.
W2
Rescore changed offerings
Open scoring.html → "Rescore" on offering. Previous scores pre-populate. Only update subcategories that moved.
W3
Update action log and assets
Mark completed actions Done in actions.html. Update statuses in gtm-assets.html. Flag new ★ priorities.
W4
Brief leadership on score changes
Portfolio overview shows trends. Hub shows current KPIs. Share offering detail page URLs for review.
Adding offerings / maintaining portal
Adding a new offering
1. scoring.html → Score an Offering → custom name → Begin Scoring · 2. Complete 13 subcats (L1 = honest baseline) · 3. Save → appears in library · 4. Add DATA entry in offering.html
Updating offering.html DATA
DATA object in offering.html: add entry with {name, group, tagline, maturityScore, catScores, subcatScores, callout, actions, aeo, gtmAssets}. Matches existing offering structure exactly.
Updating actions.html ACTIONS
ACTIONS array: add/edit entries with {id, priority, title, desc, offering, oKey, oCls, owner, status}. Use v2.0 milestone IDs (e.g. 2A.16 not 3.7).

System summary

Lighthouse is an evidence-grounded offering maturity system. Reference documents frame what good looks like. Assessors score 13 subcategory questions 1–5 using L1–L5 anchors — milestones provide the evidence base for each answer. A single weighted Maturity Score (1.00–5.00) is produced per offering, with SA/CR/DE category breakdown and 13 individual subcategory scores. Scores save to localStorage (Phase 1) or Dataverse (Phase 2). Portal pages read this data to power the Hub, Portfolio heatmap, Offering Detail, Sales View, and Action Log. The scoring session runs every 90 days. GTM Operations Director is the primary assessor. Full rebuild is possible from this document and the three source Excel/Word files.
